# 妙想 F10 压力测试结果 (2026-04-27)

## 测试环境

- 客户端: 用户机器 (Surge 开着, Mac M3)
- 服务端: `datacenter.eastmoney.com/securities/api/data/v1/get`
- 目标: `RPT_F10_EH_HOLDERNUM` (1473 页 × 500 行 = 73 万行, 最大表)
- page_size: 500
- 单 IP, 不带 SECUCODE 过滤 (全市场分页)

## 单 IP 并发探测

每组拉前 50 页 (25000 行):

| 50 | 4.17s | 12.00 | 6000 | 拥塞 |
| 100 | 3.09s | 16.17 | 8085 | 不稳 |

## 关键观察

1. **完全没有触发 rc=100 / rate-limit** — 50/100 并发也都 200 OK, 服务端没限流
2. **20 是甜蜜点** — 单 IP TCP 连接池 + 服务端处理排队, 30+ 反而拥塞
3. **没有反爬黑名单** — 跟 push2his (用户 IP 在反爬黑名单) 完全不同, datacenter 子域是公开
4. **总速度 ~26 QPS / 13K 行/秒** — 在没有 token / 不需要登录的前提下

```python
fetch_report(

```

## 全市场全报表估算

72 个 reportName 注册中, 大致分布:
- 大表 (>1000 页): 户数 / 历史财务 / 十大流通
- 中表 (100-500 页): 估值分位 / 同行排名 / 评级
- 小表 (<100 页): 单股属性类 (公司类型 / 实控人 / 基本资料)

假设平均每表 200 页, 全部并发 20 拉:
- 72 × 200 / 26 QPS = 553 秒 ≈ **9.2 分钟全部 reportName 拉一遍**

加 SECUCODE 维度的差异型 endpoint, 大部分本来就支持全市场分页 (`batch_friendly=True` 在注册表里 72/72), 不需要逐股.

## 突破并发的方法 (不需要)

- ~~多 IP / 代理池~~: 没必要, 单 IP 26 QPS 已够用
- ~~多账号~~: 接口不需要登录
- ~~分时段~~: 没限流, 任何时间都能跑

**结论**: 妙想 F10 的 datacenter 接口设计就是 "give me everything, paginated", 单 IP 20 并发够用. 不需要任何"突破并发限制"的技巧.

## 与 push2his 的对比

| 维度 | datacenter (妙想 F10) | push2his (资金流) |
|---|---|---|
| 子域 | datacenter.eastmoney.com | push2his.eastmoney.com |
| 反爬 | ✅ 用户 IP 全通 | ❌ 用户 IP 黑名单 |
| 批量友好 | ✅ 天然全市场分页 | ❌ 只支持单 secid 调用 |
| 历史深度 | 8-200 期 (按报表) | ~250 个交易日 |
| 单 IP QPS | 26 页/秒, 不限流 | RemoteDisconnected 卡死 |

教训: **不是所有 eastmoney 子域都受反爬制约**, aif10 数据中心接口可批量拉; push2 系列是行情快照, 反爬严格.
